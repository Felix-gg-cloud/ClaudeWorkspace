# 项目总览：EnglishQuestArena

## 一句话描述
RPG 游戏化的零基础英语学习系统，通过营地探索→每日主线→Boss 闯关的流程，让学习像打游戏一样有趣。

## 学习流程

```
注册 → 设置名字 → 进入 CH1
                      │
                      ▼
              ┌───────────────┐
              │  🏕️ 营地探索   │  探索地图, 遇怪→三阶段学习(学→练→结果)
              │  (Camp)       │  击败 80% 怪物解锁主线
              └───────┬───────┘
                      ▼
              ┌───────────────┐
              │  ⚔️ 每日主线   │  每天 30 道题 (seed + Auto-Drill)
              │  (Quest)      │  完成所有天数解锁 Boss
              └───────┬───────┘
                      ▼
              ┌───────────────┐
              │  💀 Boss 战    │  15 题闯关, HP对决, 连击加成
              │  (Boss)       │  通关 → 解锁下一章
              └───────┬───────┘
                      ▼
              进入下一章 CH2 → ...
              
  并行: 🏟️ SRS 副本 (每日复习 10 题, SM-2 调度)
  并行: 🌟 技能树 (技能点解锁被动技能)
  里程碑: 🔔 CEFR 等级测验 (完成一个等级所有章节后解锁)
```

## 前后端职责分工

### 前端职责 (Vue 3 + Phaser 3)

| 类别 | 具体内容 | 要点 |
|------|---------|------|
| **UI 渲染** | 10 个页面视图, 11 种题卡组件, NPC 立绘 | 所有视觉展示 |
| **游戏引擎** | Phaser 3 营地场景 (24×18 地图/碰撞/动画/迷雾/5 种怪物) | Canvas 2D 程序化纹理 |
| **交互反馈** | 连击特效, 升级动画, 粒子爆炸, 结算弹窗 | 动画和音效 |
| **音效** | 11 种 8-bit 音效 (Web Audio API) | 纯前端合成 |
| **TTS 发音** | SpeechSynthesis API (en-US / en-GB) | 浏览器内置 |
| **路由守卫** | 未登录跳转 /login | 基于 Session |
| **API 调用** | axios 封装, 统一错误处理 | withCredentials |
| **本地缓存** | 短期 UI 状态 (当前页面/展开状态等) | 非持久数据 |

**前端不做的事**:
- ❌ 不判断答案对错 (后端判卷)
- ❌ 不计算 XP/金币/等级 (后端算)
- ❌ 不决定复习队列 (后端 SRS 调度)
- ❌ 不存持久进度 (后端数据库)
- ❌ 不生成 Auto-Drill 题目 (后端生成)

### 后端职责 (Spring Boot 3)

| 类别 | 具体内容 | 要点 |
|------|---------|------|
| **认证** | Session + Cookie, 注册/登录/登出 | Spring Security |
| **数据存储** | 用户/进度/SRS/打卡 全部持久化 | PostgreSQL |
| **判卷** | 比对答案, 返回 correct + explanation | 防作弊 |
| **奖励计算** | XP/金币/等级/技能点 根据答题结果计算 | 含连击加成 |
| **SRS 调度** | SM-2 算法, 计算每词下次复习时间 | 核心学习逻辑 |
| **Auto-Drill** | seed 题不足时自动生成补充题 | 50%MCQ+30%听选+20%拼写 |
| **Boss 抽题** | 按抽题计划从已学词汇随机选题 | 每次不同 |
| **状态流转** | locked→camp→quest→boss→completed | 自动判断+流转 |
| **打卡系统** | 每日打卡, 连续天数计算 | 自动触发 |
| **Seed 导入** | 启动时读 JSON, 幂等 upsert | Flyway 建表 |

**后端不做的事**:
- ❌ 不关心 UI 如何渲染
- ❌ 不处理动画/音效
- ❌ 不管 Phaser 游戏逻辑 (地图/碰撞/特效)
- ❌ 不做 TTS 发音

## 数据流向

```
答题流程:
  用户点击选项 → 前端收集 { taskKey, answer }
                → POST /api/quest/submit
                → 后端判卷 + 计算奖励
                → 返回 { correct, explanation, xpEarned, coinsEarned, levelUp? }
                → 前端播放对应动画/音效, 显示奖励

Camp 打怪:
  用户在 Phaser 中遇怪 → Vue 弹窗三阶段学习
                      → 学习阶段 (纯前端展示)
                      → 练习阶段: 答题 → POST /api/camp/defeat { encounterId, correct, attempts }
                      → 后端更新进度 + 判断是否解锁 Quest
                      → 前端播放胜利粒子 + XP 飘字

SRS 复习:
  用户进入 Arena → GET /api/arena/session → 后端按 SM-2 选 10 个到期词
                → 前端渲染题目 → 答题 → POST /api/arena/submit { code, quality }
                → 后端更新 SRS 参数 (interval, easeFactor, nextReviewDate)
```
