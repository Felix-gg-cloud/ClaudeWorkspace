# POC-Validator — AI 验证规则清单

**版本**：v1.0  
**验证范围**：`tblOffshoring` 表格中的所有数据行  
**验证方式**：AI-only（全部规则由 Copilot Studio AI Action 执行，Power Automate Flow 不做任何预处理/过滤）

---

## 规则级别定义

| 级别 | 含义 | 处理建议 |
|------|------|---------|
| **Error** | 数据严重错误，必须修正后才能使用 | 报告中红色标注，阻断后续处理 |
| **Warning** | 数据存疑，建议人工核查 | 报告中橙色标注，不阻断处理 |
| **Info** | 信息性提示，不影响数据质量 | 报告中蓝色标注 |

---

## 全局规则（适用于所有字段）

### R-WS-ALL-001 — 前后空格检查（全局 Error）

| 属性 | 值 |
|------|----|
| **规则 ID** | R-WS-ALL-001 |
| **级别** | **Error** |
| **适用范围** | `tblOffshoring` 表中**所有文本类字段** |
| **检查逻辑** | 字段值的原始内容（AI 接收到的字符串）是否存在前导空格（leading whitespace）或尾随空格（trailing whitespace） |
| **触发条件** | `value !== value.trim()`（即原始值与去除首尾空格后的值不一致） |
| **错误消息模板** | `字段 "{field}" 第 {RowIndex} 行存在前导或尾随空格，请删除多余空格后重试。` |
| **说明** | Power Automate Flow 不对数据做任何 trim 预处理，所有空格原样传给 AI，由 AI 检测并报告。 |

**示例**：

| 场景 | 原始值 | 是否触发 |
|------|--------|---------|
| 值前有空格 | `" IBM"` | ✅ 触发（Error） |
| 值后有空格 | `"IBM "` | ✅ 触发（Error） |
| 值前后均有空格 | `" IBM "` | ✅ 触发（Error） |
| 值中间有空格 | `"IBM China"` | ❌ 不触发 |
| 正常值 | `"IBM"` | ❌ 不触发 |
| 空字符串 | `""` | ❌ 不触发（由必填规则单独处理） |

## 规则优先级与执行顺序

AI 在检查时应按以下顺序执行规则，同一字段可触发多条规则：

1. **R-WS-ALL-001**（全局，最先执行）：检测所有文本字段的首尾空格
2. 必填规则（`*-001` / `*-002` 中属于必填检查的）
3. 格式规则
4. 逻辑规则（跨字段验证）
5. 范围/枚举规则
6. Info 提示类规则（最后执行）

> 即使某字段已触发 R-WS-ALL-001，其他规则仍继续检查，不跳过。

## issues[] 数组字段规范

AI 返回的 `issues` 数组中，每个元素必须包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `RowIndex` | Integer | **必填** | 对应源数据的 Excel 行号（从 2 开始） |
| `field` | String | 必填 | 触发问题的字段名 |
| `ruleId` | String | 必填 | 触发的规则 ID，如 `R-WS-ALL-001` |
| `level` | String | 必填 | `Error`、`Warning` 或 `Info` |
| `message` | String | 必填 | 对用户友好的错误描述（中文） |
| `actualValue` | String | 推荐 | 触发问题的实际字段值（原始值，不做处理） |

> **`RowIndex` 为必填字段**，AI 必须在每条 issue 中返回该值，以便 Flow 和最终报告能追溯到 Excel 中的具体行。
