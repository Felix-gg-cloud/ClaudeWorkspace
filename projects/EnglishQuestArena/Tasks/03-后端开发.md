# 阶段三：后端开发

## 状态：⬜ 未开始 | 依赖于阶段二完成

## 目标
Spring Boot 后端 API，管理用户、课程进度、SRS 复习、等级测验。
后端是**数据权威**：所有判卷、奖励计算、SRS 调度、进度状态流转由后端负责。

---

## 技术栈
- Java 17 + Spring Boot 3.x
- Maven
- Spring Security (Session + Cookie, 非 JWT)
- Spring Data JPA
- H2 (开发) / PostgreSQL (生产)
- Flyway (数据库迁移)

---

## 核心职责（后端独占）

> 以下逻辑**只在后端实现**，前端不做判断，只负责展示后端返回的结果。

| 职责 | 说明 | 前端对应 |
|------|------|---------|
| **判卷** | 比对答案是否正确，返回 correct/wrong + explanation | 前端提交答案 → 显示结果 |
| **XP/金币计算** | 根据答题结果+连击+技能加成计算奖励 | 前端展示后端返回的奖励数值 |
| **等级升级** | XP 达到阈值自动升级，返回新等级+技能点 | 前端播放升级动画 |
| **章节状态流转** | locked→camp→quest→boss→completed | 前端查询当前状态, 渲染对应页面 |
| **SRS 调度** | SM-2 算法计算下次复习时间 | 前端只显示后端推送的复习题 |
| **Auto-Drill 出题** | seed 题不足时自动生成 MCQ/听选/拼写题 | 前端无感知, 当普通题渲染 |
| **Boss 抽题** | 按抽题计划从题池随机选题 | 前端只渲染后端返回的题目 |
| **打卡记录** | 每日完成首课自动打卡, 计算连续天数 | 前端展示打卡日历 |
| **每日限制** | Boss 每日 1 次, CEFR 测验每日限次 | 前端提示剩余次数 |

CREATE TABLE srs_records (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),

    ease_factor DECIMAL(4,2) DEFAULT 2.5,  -- SM-2 易度因子
    interval_days INT DEFAULT 1,            -- 当前间隔天数
    repetitions INT DEFAULT 0,              -- 连续正确次数
    next_review_date DATE NOT NULL,         -- 下次复习日期
    total_reviews INT DEFAULT 0,
    total_correct INT DEFAULT 0,
    last_reviewed_at TIMESTAMP,
    UNIQUE(user_id, content_item_code)

CREATE TABLE boss_attempts (
    id BIGSERIAL PRIMARY KEY,

## 核心模块 & API

### 模块 1: 用户与认证
```
POST   /api/auth/register          → 注册
POST   /api/auth/login             → 登录 (Session + Cookie)
POST   /api/auth/logout            → 登出
GET    /api/users/me               → 获取当前用户信息
PATCH  /api/users/me               → 修改 displayName, ttsVoice
```

### 模块 2: 课程内容查询
```
GET    /api/chapters               → 章节列表 + 当前用户各章进度
GET    /api/lessons?chapter=xxx    → 指定章节的课时列表
GET    /api/camp/map?chapter=xxx   → 营地地图数据 (含怪物击败状态)
```

### 模块 3: 主线 Quest
```
GET    /api/quest/today            → 今日课程 + 题目列表 (seed + auto-drill) + 进度
POST   /api/quest/submit           → 提交答案 { taskKey, answer }
                                     返回 { correct, explanation, xpEarned, coinsEarned, levelUp? }
```

### 模块 4: 营地 Camp
```
POST   /api/camp/defeat            → 提交怪物击败 { encounterId, correct, attempts }
POST   /api/camp/open-chest        → 开宝箱 { encounterId }
```

### 模块 5: SRS 副本 Arena
```
GET    /api/arena/session          → 获取今日 SRS 复习题 (10 道, SM-2 调度)
POST   /api/arena/submit           → 提交复习答案 { contentItemCode, quality }
                                     quality: 0-5 (SM-2 评分)
                                     返回 { correct, nextReviewDate, interval }
```

### 模块 6: Boss 战
```
GET    /api/boss/:code             → 获取 Boss 配置 + 抽题后的题目列表
POST   /api/boss/answer            → 逐题提交 { bossCode, taskKey, answer }
                                     返回 { correct, bossDamage, playerDamage, combo }
POST   /api/boss/finish            → 战斗结束结算 { bossCode, victory }
                                     返回 { xpEarned, chapterCompleted, nextChapterUnlocked }
```

### 模块 7: 等级测验 CEFR Exam
```
GET    /api/cefr-exam/status       → 各等级测验解锁状态 + 今日剩余次数
GET    /api/cefr-exam/start/:level → 开始测验 (随机抽题)
POST   /api/cefr-exam/submit       → 提交测验结果
                                     返回 { score, passed, cefrLevelUp? }
```

### 模块 8: 技能树
```
GET    /api/skill-tree             → 全部节点 + 已解锁状态
POST   /api/skill-tree/unlock/:code → 消耗技能点解锁
```

### 模块 9: 打卡与统计
```
POST   /api/checkin                → 每日打卡 (通常由完成课程自动触发)
GET    /api/checkin/calendar?month=YYYY-MM → 打卡日历
```

## SRS SM-2 算法伪代码

```
function updateSRS(record, quality):
    // quality: 0=完全忘记, 3=勉强记住, 5=完全掌握
    if quality >= 3:  // 正确
        if record.repetitions == 0:
            record.interval = 1
        elif record.repetitions == 1:
            record.interval = 6
        else:
            record.interval = round(record.interval * record.easeFactor)
        record.repetitions += 1
    else:  // 错误
        record.repetitions = 0
        record.interval = 1
    
    record.easeFactor = max(1.3,
        record.easeFactor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)))
    record.nextReviewDate = today + record.interval days
```

## Auto-Drill 自动出题规则

1. 取 seed tasks (按 orderIndex 排序)
2. 若 tasks.size < targetTaskCount (默认 30) 且 autoDrillEnabled=true:
   - 收集本 lesson 关联的所有 contentItems (去重)
   - 只选 type=WORD 的 contentItems
   - 按比例生成:
     - 50%: MCQ (词义选择: "Choose the meaning of 'word'")
     - 30%: MCQ (听音选词: ttsTextEn=word, 3-4 个同章节词作选项)
     - 20%: SPELLING (输入拼写: ttsTextEn=word)
3. Auto tasks code 以 `AUTO_` 前缀, xpReward=2~4, goldReward=0
4. Auto tasks 不写入 Task 表, 运行时生成; 但答题结果写入 task_progress
