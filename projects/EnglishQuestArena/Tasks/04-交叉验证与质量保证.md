# 阶段四：交叉验证 & 质量保证

## 状态：⬜ 未开始 | 依赖于阶段二+三完成

## 目标
对照 CEFR 官方标准，验证每个等级的内容覆盖度和准确性，确保真正能帮用户达到对应等级。

---

## 前置条件
- [x] 阶段一完成: 前端所有题型可渲染
- [ ] 阶段三完成: 后端 API 可运行, 前后端联调通过
- [ ] 阶段二完成: Seed 数据已编写 (PRE_A1 至少 100+ ContentItems)

---

## 验证清单

### 1. 词汇覆盖度验证
- [ ] 对照 English Profile EVP 词汇表，检查 PRE_A1 词汇是否全覆盖
- [ ] 对照 English Profile EVP 词汇表，检查 A1 词汇是否全覆盖
- [ ] 对照 Oxford 3000 A1 部分交叉验证
- [ ] 统计遗漏词汇，补充到 content_items
- [ ] 验证每个词的 IPA 音标正确
- [ ] 验证每个词至少有 2 个有效 distractors

### 2. 语法覆盖度验证
- [ ] 对照 Cambridge A1 大纲语法点清单
- [ ] 每个语法点至少有:
  - 1 个 FLASHCARD (认知)
  - 2 个 MCQ (理解)
  - 1 个 FILL_BLANK (运用)
  - 1 个 WORD_ORDER (造句)
- [ ] 检查语法点的教学顺序是否合理（先简后难）

### 3. 场景/话题覆盖度验证
- [ ] 对照 CEFR A1 Can-do Statements:
  - 能进行简单自我介绍
  - 能理解和使用日常表达
  - 能进行简单购物对话
  - 能问路和理解简单方向
  - 能填写简单表格
  - ... (逐项检查)

### 4. 等级测验质量验证
- [ ] 测验题覆盖该等级所有核心知识点
- [ ] 题型分布合理 (MCQ 30% + FILL_BLANK 20% + SPELLING 15% + LISTEN_PICK 15% + WORD_ORDER 10% + SITUATION_PICK 10%)
- [ ] 难度适中 (通过率目标 60-80%)

### 5. 内容准确性验证
- [ ] 所有例句语法正确
- [ ] 翻译准确无误
- [ ] TTS 发音文本正确
- [ ] 选项无歧义（错误选项不能也是正确答案）

### 6. SRS 复习效果验证
- [ ] SM-2 参数合理 (第1次=1天, 第2次=6天, 第3次=15天...)
- [ ] 每日复习量 10 题左右, 不过多不过少
- [ ] 答错的词快速回到队列 (interval 重置为 1 天)

### 7. 端到端学习路径体验测试
- [ ] 从零开始模拟完整学习路径 (注册 → Camp → Quest → Boss → 下一章)
- [ ] 检查难度曲线是否平滑
- [ ] 检查每日任务量是否合理（~30 分钟）
- [ ] Auto-Drill 生成题质量抽查
- [ ] Boss 战体验: 3-5 次通关是否合理

### 8. 技术验证
- [ ] 前后端 API 所有接口联调通过
- [ ] 多用户并发测试 (至少 5 个模拟用户)
- [ ] 数据一致性检查 (前端显示 vs 后端数据库)
- [ ] Seed 重复导入幂等性验证

---

## 验证工具
- 脚本: 统计 content_items 覆盖率 vs 参考词表
- 脚本: 检查 JSON 引用完整性 (distractors, contentItemCodes)
- 手动: 走查每个章节的教学流程
- 手动: 邀请 2-3 个零基础用户实际使用

## 修复流程
1. 发现遗漏 → 补充到 content_items + tasks (阶段二)
2. 发现错误 → 修正 seed 数据
3. 难度不合理 → 调整章节编排或题目
4. SRS 参数不合理 → 调整 SM-2 初始值
5. 重新验证直到通过
